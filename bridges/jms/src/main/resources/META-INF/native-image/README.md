# GraalVM Native Image Configs

These files are used during `cargo xtask build-jms-bridge` to produce a correct native binary.

The JSON files (`reflect-config.json`, `resource-config.json`, `proxy-config.json`, `jni-config.json`)
are generated by the GraalVM tracing agent and committed to the repo.

## When to regenerate

Regenerate these configs whenever any of these dependencies have version bumps:
- `org.apache.activemq:activemq-client`
- `org.apache.activemq:artemis-jms-client-all`
- `org.messaginghub:pooled-jms`

## How to regenerate

See: `docs/superpowers/specs/2026-04-03-jms-bridge-dx-design.md` → "Generation process"

Short version:
1. `cd bridges/jms && ./gradlew build --no-daemon`
2. Start ActiveMQ on port 61616 and Artemis on port 61617 via Docker
3. `java -agentlib:native-image-agent=config-output-dir=src/main/resources/META-INF/native-image -jar build/quarkus-app/quarkus-run.jar`
4. Exercise all JMS integration tests (including error paths)
5. Kill the bridge process and stop Docker containers
6. Commit the updated config files
7. Run `cargo xtask build-jms-bridge` to verify native build succeeds
