# schemas/dsl/

JSON Schema for the rust-camel route DSL.

## Scope

The schema documents the **public SDK surface**: a route file is `{routes: [...]}` plus an optional `$schema` key. Templates (`templates: [...]` and `templated_routes: [...]` in the runtime AST) are excluded — they are internal machinery that uses a dynamic YAML value type not representable in JSON Schema.

## Files

- `route-schema.json` — generated by `cargo xtask schema`.

## URL convention

- **Main:** `https://raw.githubusercontent.com/kennycallado/rust-camel/main/schemas/dsl/route-schema.json`
- **Tagged (v1.0+):** pin to the release tag, e.g. `.../v1.0.0/schemas/dsl/route-schema.json`.

## Usage

Add `"$schema"` to your route file:

```json
{
  "$schema": "https://raw.githubusercontent.com/kennycallado/rust-camel/main/schemas/dsl/route-schema.json",
  "routes": [...]
}
```

The parser silently ignores the key (it does not validate the URL). Editors (VS Code, JetBrains) and SDKs consume it for autocomplete and validation.

## Regenerating

`cargo xtask schema` regenerates all schema artifacts. `cargo xtask schema --check` is a CI gate that fails on drift.

## Versioning

Post-1.0, schema version is decoupled from crate version. Breaking schema changes will bump the schema URL or `$id`.
